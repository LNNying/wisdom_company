手机管理系统数据库设计

1. tp_Role 角色表
   id         int primary key 自增
   role_name  varchar(40) not null  -----  管理员，部门经理，员工
   

2. tp_Manger 员工表
   id            int not null primary key 自增
   manger_name   varchar(40)  not null
   role_id       int not null                员工与角色  一对多    员工 belongsTo   角色  hasMany
   department_id  int not null               员工与部门  一对多    员工 belongsTo   部门  hasMany
   phone         varhchar(255) not null
   email         varchar(255)
   
   
3. tp_Department 部门表
   id            int not null primary key 自增
   dep_name      varchar(40) not null   ---- 销售部门，售后部门
   dep_desc      varchar(255)
   

4. tp_Product   产品表
   id 			int not null primary key 自增
   pro_name     not null varchar(40) 产品名称
   pro_all_num  int not null   产品总数量
   pro_num      int 默认为0    剩余数量   
   pro_all_price    int not null   产品销售价格
   pro_price    int not null   产品进价
   pro_desc     varchar(255)        产品描述
   
   
5.tp_Servse     反馈表
  id			int not null primary key 自增
  cus_id        int not null         客户姓名
  ser_desc		varcahr(225)         反馈内容
  pro_id        int not null         产品id
  ser_date		datatime 后端自动生成   反馈时间
  
  
6.tp_Customer   客户表
  id            int not null primary key 自增
  cus_name      varchar(40) not null  客户姓名
  cus_phone     varchar(20) not null  客户联系方式
  cus_address   varchar(40)           客户地址
  
产品与订单 是一对多  
7.tp_Order      订单表
  id            int not null primary key 自增
  order_id      varchar(20) not null   订单号  前端自动生成合格单号
  cus_id        int not null           客户id
  pro_id        int not null           产品id
  pro_num       int not null           产品数量
  pro_price     int not null           该订单的总价格
  
  
8.tp_System     系统配置表
  sys_title    varchar(40) not null    网页标题
  sys_footer   varchar(40) not null    网页页脚
